AssetNote — Claude Code 専用仕様書（claude.md）

このファイルは AssetNote を Claude Code で開発・編集するための、 アプリ専用の 設計書 / UI 仕様 / コーディングルール です。 PocketPlus や他アプリの要素は含めず、AssetNote のみに最適化しています。

Claude Code はこの claude.md を常に参照し、 AssetNote の世界観・構造・命名・UI を統一した状態でコードを生成します。

1. AssetNote の世界観・デザイン方針

■ 全体テーマ

Apple 風の ミニマル・情報密度高め

dark、lightモードに対応

色は控えめ、Primary は青系

カード UI を中心に構成

■ フォント

iOS 系（San Francisco 風）

太字は最小限

■ 複数端末対応
ストアアプリ公開（googleストア、applestore）を予定しているので、
端末によってデザインが崩れることがないように考慮します。


2. Claude Code の動作ルール（AssetNote 専用）

Claude Code は以下を厳守する：

claude.md を常に参照してコードを生成する

AssetNote の UI 世界観（Apple 風ミニマル）を維持する

完全置き換えコードを返す（部分修正しない）

不要なロジックは削除してよい

改善提案を積極的に行う

Widget 分離を推奨する

迷ったらミニマル・シンプルを優先

1. DB 操作の許可方針
- このシステムでは、データベースへの追加・削除を行ってよい。
- モデルは必要に応じて、DB のレコード追加・削除を提案・実行指示してよい。

2. SQL での追加操作の指示を許可
- モデルは、ユーザーの要求に応じて SQL を用いたデータ追加方法を説明してよい。
- 必要であれば INSERT 文などの SQL を生成してよい。
### Database Operations
- このシステムでは、データベースに対する「追加」「削除」操作を許可する。
- モデルは、ユーザーの要求に応じてデータベース操作を提案してよい。
- SQL によるデータ追加（INSERT）、削除（DELETE）などのクエリを生成してよい。
- 必要に応じて、SQL を用いたデータ操作手順を説明してよい。
